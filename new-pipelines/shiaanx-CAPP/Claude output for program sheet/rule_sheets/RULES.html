Sheet 01 — Feature Classification Rules

Decision Priority

  1. seed_type branches: background → background; plane/plane_feature → pocket vs planar_face; slot → slot; boss → boss; bore → bore tree
  2. bore: len(radii) >= 2 → counterbore vs large_bore (max/min radius vs thresholds)
  3. bore: single radius, radius >= large_bore_radius_mm → large_bore
  4. bore: single radius, face_count >= 2 → through_hole
  5. bore: single face → ddr = depth / (2*radius); ddr <= single_face_through_hole_ddr_max → through_hole (medium confidence) else blind_hole (low)
  6. append _angled when is_principal_axis is false (except unknown_bore)

Thresholds

ParameterValueUnitRole
large_bore_radius_mm10.0mmRadius above which a single-step bore is large_bore; also used in multi-radius bore max-radius check.
single_face_through_hole_ddr_max0.5ratioIf depth / (2*r) <= this, treat single-face bore as through_hole (thin arc / exit not in cluster).
pocket_max_area_mm2500.0mm2Plane cluster with perpendicular walls must have floor area below this to be pocket vs planar_face.
drill_max_radius_mm8.0mmIf smallest radius in a multi-radius bore >= this, classify whole feature as large_bore (cannot drill smallest step).
pocket_max_perp_walls8countMax perpendicular wall count for a plane cluster to be classified as pocket. Real machined pockets have 2-6 walls. Stock or datum faces with many adjacent walls (e.g. 15) are excluded by this cap.

Min perpendicular walls: 2  |  High-confidence: 3

⚠ Notes
  • chamfer and tapped_hole are not produced by classify_cluster() today; MFCAD supervision uses 07_label_taxonomy.json.
  • large_bore_angled is produced by classifier but select_process() only branches on 'large_bore' without suffix — verify dispatcher if angled large bores appear.

Spring Pass

A spring pass is a repeat of the finish pass at zero additional DOC. It recovers elastic deflection of the tool and workpiece that occurred on the first finish pass. For aluminium facing, the strategy sheet marks Is_Finishing=TRUE rules with Spring_Pass=TRUE. In practice: emit a third face_mill step with pass_type=SPRING, identical parameters to FINISH, zero stock_to_leave. Not yet implemented in process_selection.py — tracked for future addition.

Sheet 05 — Setup Planning Rules

WCS: G54 → G55 → G56 → G57 → G58 → G59

Corner-zero tolerance: 2% — Treat bounding box min as 'at CAD origin' if |min| < fraction × (max − min) for that axis pair used in the setup plane

Grouping Algorithm

  1. Collect machinable clusters: not background, machine_type not none, has feature_axis
  2. Planar and no-axis clusters assigned separately to best existing setup
  3. Group by exact same-direction feature_axis using _are_same_direction
  4. spindle_dir = normalize(-rep_axis)
  5. Principal groups: setup_type principal; non-principal: setup_type angled + rotation_from_default

Stock: Setup 1 = raw_billet, later = previous_setup

⚠ Notes
  • Full WCS coordinate formulas per spindle direction remain in _compute_wcs_origin(); this sheet captures thresholds and intent only.
  • LIVE (applied at import): vmc_convention.default_spindle_direction_cad, axis_parallel_tolerance, wcs.sequence, wcs_origin_corner_heuristic.fraction_of_dimension, stock_state.all_part_faces.
  • DOCUMENTATION ONLY (not yet globals): grouping_algorithm, axis_group_sort, setup_order_within_group, machinable_filters — these are fixed logic in plan_setups().
